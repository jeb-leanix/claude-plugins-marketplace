# Contributing to LeanIX Team Marketplace

## For Plugin Users

Just clone the **`built` branch** and start using plugins - no setup needed!

```bash
git clone -b built --recurse-submodules https://github.com/jeb-leanix/claude-plugins-marketplace.git
```

## For Plugin Developers

### Adding a New Plugin

1. **Create plugin repo** (use [plugin-template](https://github.com/jeb-leanix/claude-plugin-template))

2. **Add as submodule** to marketplace:
   ```bash
   cd claude-plugins-marketplace
   git submodule add https://github.com/jeb-leanix/your-plugin.git plugins/shared/your-plugin
   # or for team-specific:
   git submodule add https://github.com/jeb-leanix/your-plugin.git plugins/team-takeoff/your-plugin
   ```

3. **Register in `registry.json`**:
   ```json
   {
     "id": "your-plugin",
     "name": "Your Plugin Name",
     "version": "1.0.0",
     "team": "shared",
     "visibility": "public",
     "description": "What your plugin does",
     "path": "plugins/shared/your-plugin",
     "repository": "https://github.com/jeb-leanix/your-plugin.git",
     "category": "productivity",
     "tags": ["tag1", "tag2"]
   }
   ```

4. **Update marketplace.json**:
   ```bash
   ./sync-marketplace.sh
   ```

5. **Test locally**:
   ```bash
   ./build-plugins.sh
   # Add to your Claude settings and test
   ```

6. **Commit and push**:
   ```bash
   git add .
   git commit -m "Add your-plugin to marketplace"
   git push origin main
   ```

7. **CI builds automatically** → Plugin available in `built` branch!

### Updating an Existing Plugin

**Option A: Update submodule reference** (if plugin repo was updated)
```bash
cd plugins/shared/your-plugin
git pull origin main
cd ../../..
git add plugins/shared/your-plugin
git commit -m "Update your-plugin to latest"
git push origin main
```

**Option B: Edit plugin code directly**
```bash
cd plugins/shared/your-plugin
# Make changes
git add .
git commit -m "Fix: your changes"
git push origin main
cd ../../..
git submodule update --remote plugins/shared/your-plugin
git add plugins/shared/your-plugin
git commit -m "Update your-plugin submodule"
git push origin main
```

### Branch Strategy

- **`main` branch**: Source code only, no build artifacts
  - Contains submodule references
  - `.gitignore` excludes `build/` and `node_modules/`
  - Developers work here

- **`built` branch**: Auto-generated by CI
  - Contains all build artifacts (`build/`, `node_modules/`)
  - Updated automatically on every push to `main`
  - Users clone this branch

### CI/CD Pipeline

GitHub Actions (`.github/workflows/build-plugins.yml`) automatically:
1. Checks out `main` with submodules
2. Runs `./build-plugins.sh`
3. Commits built artifacts to `built` branch
4. Pushes to remote

**Never manually commit to `built` branch!**

### Testing Changes

Before pushing:
```bash
# Build locally
./build-plugins.sh

# Test in Claude Code
# Edit ~/.claude/settings.json to point to your local repo
{
  "extraKnownMarketplaces": {
    "leanix-team-marketplace": {
      "source": {
        "source": "directory",
        "path": "/absolute/path/to/your/local/clone"
      }
    }
  },
  "enabledPlugins": {
    "your-plugin@leanix-team-marketplace": true
  }
}

# Reload plugins
/reload-plugins
```

## Plugin Requirements

### File Structure
```
your-plugin/
├── .mcp.json or plugin.json  # Plugin metadata
├── package.json               # npm dependencies
├── tsconfig.json             # TypeScript config
├── src/                      # Source code
│   └── index.ts
├── commands/                 # Command documentation
│   └── your-command.md
└── README.md                 # Usage documentation
```

### Metadata (`.mcp.json`)
```json
{
  "name": "your-plugin",
  "version": "1.0.0",
  "description": "Short description",
  "entrypoint": "build/index.js",
  "runtime": "node",
  "commands": [
    {
      "name": "your-command",
      "description": "What it does",
      "usage": "/your-command [options]"
    }
  ]
}
```

### Build Script (`package.json`)
```json
{
  "scripts": {
    "build": "tsc"
  }
}
```
